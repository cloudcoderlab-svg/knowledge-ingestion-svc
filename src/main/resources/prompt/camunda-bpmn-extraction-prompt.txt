=== CAMUNDA BPMN ELEMENT MAPPINGS ===

**Process Structure:**
- <bpmn:process id="X" name="Y"> → businessFlows.flowName = "Y"
- <bpmn:documentation> → Extract business purpose and rationale
- <bpmn:extensionElements> → Camunda-specific configurations

**Tasks:**
- <bpmn:userTask> → businessFlow.steps with actor from assignee/candidateGroups
- <bpmn:serviceTask> → solutionComponents (service integrations)
- <bpmn:businessRuleTask> → businessRules
- <bpmn:scriptTask> → solutionComponents with componentType="script"
- <bpmn:manualTask> → businessFlow.steps with manual activity
- <bpmn:sendTask> / <bpmn:receiveTask> → solutionComponents (messaging)

**Gateways:**
- <bpmn:exclusiveGateway> → businessDecisions (XOR - choose one path)
- <bpmn:parallelGateway> → businessFlow.steps (AND - parallel execution)
- <bpmn:inclusiveGateway> → businessDecisions (OR - one or more paths)
- <bpmn:eventBasedGateway> → businessDecisions (event-driven routing)

**Events:**
- <bpmn:startEvent> → businessFlows.trigger
- <bpmn:endEvent> → businessFlows.outcome
- <bpmn:intermediateThrowEvent> → businessFlow.steps (signal/message)
- <bpmn:intermediateCatchEvent> → businessFlow.steps (waiting)
- <bpmn:boundaryEvent> → businessRules (exception handling)
- <bpmn:timerEvent> → businessFlow.steps with timing constraints

**Lanes & Pools:**
- <bpmn:lane name="X"> → businessRoles.roleName = "X"
- <bpmn:participant> → businessRoles or external system
- <bpmn:collaboration> → relationships between processes

**Data:**
- <bpmn:dataObject> → dataModels or businessTerms
- <bpmn:dataStoreReference> → deploymentResources (database)
- <bpmn:property> → process variables

**Camunda Extensions:**
- <camunda:formData> → UI forms for user tasks
- <camunda:executionListener> → Event handlers
- <camunda:taskListener> → Task lifecycle handlers
- <camunda:inputOutput> → Data mappings
- <camunda:connector> → External system integrations

**Sequence & Message Flows:**
- <bpmn:sequenceFlow> → Determines steps.nextStep and ordering
- <bpmn:conditionExpression> → businessDecisions.decisionCriteria
- <bpmn:messageFlow> → relationships with relationshipType="sends_message_to"
