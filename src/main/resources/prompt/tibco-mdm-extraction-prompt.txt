=== TIBCO MDM ELEMENT MAPPINGS ===

**Repository Structure:**
- <Repository name="X"> → Container for entities (maps to dataModels collection)
- <Entity name="Y"> → Business entity definition (maps to dataModels.modelName)
- <Field name="F" type="T"> → Entity attribute (maps to dataModels.fields)

**Validation & Rules:**
- <Validation> → Business rule with ruleType="validation"
- <DataQualityRule> → Business rule with ruleCategory="quality"
- <CalculatedField> → Business rule with ruleType="calculation"
- <ConditionalValidation> → Business rule with complex conditions

**Relationships:**
- <Relationship type="parent-child"> → knowledge_relationships with relationshipType="parent_of"
- <Relationship type="lookup"> → dataModels.fields with foreign key constraint
- <Relationship type="association"> → knowledge_relationships with relationshipType="associated_with"

**Workflows:**
- <WorkflowTemplate> → businessFlows (data stewardship workflows)
- <WorkflowStep> → businessFlow.steps
- <WorkflowTransition> → determines nextStep
- <WorkflowRole> → businessRoles for stewards/approvers

**Master Data Concepts:**
- <MasterRecord> → dataModels with masterData=true
- <GoldenRecord> → dataModels with masterData=true
- <LookupTable> → dataModels with referenceData=true
- <CrossReference> → knowledge_relationships with relationshipType="same_as"

**Business Context:**
- <Description> → Extract into businessDefinition
- <BusinessName> → Extract into businessName
- <DataOwner> → Extract into businessOwner
- <DataSteward> → Extract into businessRoles
- <SensitivityLevel> → Maps to dataSensitivity
- <QualityMetric> → Maps to businessMetrics

=== EXTRACTION INSTRUCTIONS ===

1. **Entity Extraction:**
   - Extract ALL <Entity> elements as dataModels
   - Use Entity @name for modelName
   - Use <BusinessName> or <Description> for businessName and businessDefinition
   - Set masterData=true if entity is marked as master data
   - Extract <DataOwner> to businessOwner field

2. **Field Extraction:**
   - Extract ALL <Field> elements within entities
   - Capture field @name, @type, @required attributes
   - Extract <Description> and <BusinessName> if present
   - Extract <DefaultValue>, <AllowedValues> as constraints
   - Extract <BusinessExamples> if present

3. **Validation Rules:**
   - Extract ALL <Validation> elements as businessRules
   - Parse condition expressions into condition field
   - Extract error messages into businessImpact
   - Use <Priority> for priority field
   - Extract <BusinessRationale> if present

     * Validation regex patterns, formats, constraints
     * Service or API names that perform validation
     * Error codes and error message templates
     * Data type constraints, range validations
     * Calculation formulas with actual syntax
   - **NEW: validationCriteria** - Create test scenarios:
     * Example valid inputs and expected results
     * Example invalid inputs and expected error codes/messages
     * Edge cases and boundary conditions

4. **Relationships:**
   - Extract ALL <Relationship> elements
   - Map source and target entity names
   - Use relationship @type for relationshipType
   - Extract cardinality and cascade rules

5. **Workflows:**
   - Extract <WorkflowTemplate> as businessFlows
   - Parse all workflow steps in sequence
   - Extract role assignments for each step
   - Capture approval/rejection paths
   - **NEW: For each workflow step, extract technical details:**
     * **technicalDetails** - Service calls, API endpoints, database operations, rulebase references (e.g., "rbCheckValidation"), transformations, error handling
     * **inputParameters** - Parse <ActualParameters> or <InputData> into JSON format: {"paramName": {"type": "String", "required": true, "description": "..."}}
     * **outputParameters** - Parse output variables into JSON format: {"resultName": {"type": "Boolean", "description": "..."}}

6. **Data Quality:**
   - Extract match rules, merge rules, survivorship rules
   - Map to businessRules with appropriate ruleType
   - Extract data quality dimensions as businessMetrics
