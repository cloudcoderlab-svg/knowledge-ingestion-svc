=== PEGA ELEMENT MAPPINGS ===

**Case Structure:**
- <pega:CaseType> → businessCapabilities (case types represent capabilities)
- <pega:Stage> → businessFlow.steps (case lifecycle stages)
- <pega:Process> / <pega:Flow> → businessFlows
- <pega:Step> → businessFlow.steps

**Flow Elements:**
- <pega:Assignment> → businessFlow.steps with actor (user task)
- <pega:Connector> → solutionComponents (integration connectors)
- <pega:Subprocess> → nested businessFlows or relationships
- <pega:Decision> → businessDecisions
- <pega:Utility> → solutionComponents (automation activities)

**Decision & Rules:**
- <pega:DecisionTable> → businessRules with structured conditions
- <pega:DecisionTree> → businessRules with hierarchical logic
- <pega:When> conditions → businessDecisions or businessRules
- <pega:Validate> → businessRules with ruleType="validation"

**Participants & Roles:**
- <pega:Participant> → businessRoles
- <pega:WorkBasket> → businessRoles (shared work queues)
- <pega:WorkGroup> → businessRoles (team assignments)

**Data:**
- <pega:DataTransform> → businessRules with ruleType="calculation"
- <pega:DataPage> → dataModels (reusable data sources)
- <pega:Property> → dataModels.fields
- <pega:DataClass> → dataModels

**Service Integrations:**
- <pega:Connect-REST> → solutionComponents (REST integration)
- <pega:Connect-SOAP> → solutionComponents (SOAP integration)
- <pega:Connect-SQL> → solutionComponents (database integration)
- <pega:Connect-File> → solutionComponents (file operations)

**UI & Forms:**
- <pega:Section> → UI components (can be solution components)
- <pega:Harness> → UI layout patterns

**SLA & Metrics:**
- <pega:ServiceLevel> → businessMetrics or businessPolicies
- <pega:Goal> → businessMetrics
- <pega:Deadline> → businessPolicies with enforcement

=== EXTRACTION INSTRUCTIONS ===

1. **Case Type Extraction:**
   - Extract <pega:CaseType> as businessCapabilities
   - This represents a business capability (e.g., "Loan Application", "Customer Onboarding")
   - Extract stages as high-level workflow phases

2. **Flow Extraction:**
   - Extract <pega:Process> and <pega:Flow> as businessFlows
   - Use flow name for flowName
   - Extract purpose from description or documentation
   - Map all steps in sequence

3. **Step Extraction:**
   - Extract all flow steps (Assignment, Decision, Connector, Utility)
   - For Assignment steps: extract actor from participant/workbasket
   - For Decision steps: map to businessDecisions
   - For Connector steps: map to solutionComponents

4. **Decision Extraction:**
   - Extract DecisionTable and DecisionTree as businessRules
   - Parse condition columns and outcome columns
   - Extract business rationale from rule descriptions
   - Map priority/precedence

5. **Role Extraction:**
   - Extract participant names, workbaskets, workgroups
   - Map to businessRoles with appropriate roleType
   - Extract responsibilities from descriptions

6. **Integration Extraction:**
   - Extract all Connect-* elements as solutionComponents
   - Parse endpoint URLs, authentication methods
   - Extract data mappings for request/response

7. **Data Extraction:**
   - Extract DataTransform logic as businessRules
   - Extract DataPage definitions as dataModels
   - Parse Property definitions and their types

8. **SLA Extraction:**
   - Extract ServiceLevel definitions as businessMetrics or policies
   - Parse goal and deadline timeframes
   - Extract escalation actions
